# V5.2 Design: Clipped-OKLab Repulsion Energy

**Date:** 2026-04-10
**Spec:** `Specs/Facette_algorithm_v5.2_amendment.md`
**Approach:** C — single ForceComputer with blended inner loop

---

## Problem

With vivid wide-hue seeds (high gamma), the radial lift pushes the hull far beyond sRGB. After inverse transform and gamut clip, particles cluster at low-chroma positions. The root cause: Riesz repulsion in lifted space is a proxy for output separation, and the proxy diverges when clipping is significant.

## Change Summary

Replace lifted-space Riesz energy with a β-blended energy that transitions from lifted-space distances to clipped-OKLab distances during the p ramp.

**What changes:**
- `energy.ts` — repulsion energy computed on blended lifted + clipped distances
- `types.ts` — `ForceComputer` gains β parameter; `AnnealingSchedule` gains `getMetricBlend`
- `optimization.ts` — passes β from schedule to force computer

**What stays the same:**
- Hull construction, face atlas, surface navigation
- Gamut penalty E_gamut (retained, complements clipped repulsion)
- Initialization, convergence criteria, output pipeline
- All parameters (gamma, r_s, R, spread, vividness)

---

## Design

### 1. Annealing Schedule — β ramp

`AnnealingSchedule` gains:

```ts
getMetricBlend(iteration: number): number  // β ∈ [0, 1]
```

β ramps linearly 0→1 over the same range as the p ramp (0 to `rampIterations`). After the ramp, β = 1 (pure clipped energy).

`AnnealingScheduleOptions` gains `metricBlendEnd?: number` defaulting to the resolved value of `rampIterations` (which itself defaults to 1000). Both are independently settable.

`ForceComputer.computeForcesAndEnergy` gains a 4th parameter `beta: number`.

### 2. Blended Repulsion Energy

The pairwise loop computes both metrics in one pass per pair:

1. **Lifted distance** `d_L = ||p_i - p_j||` — analytical (existing)
2. **Clipped distance** `d_C = ||q_i - q_j||` where `q_i = clip(T⁻¹(p_i))` — new

Blended energy per pair:

```
E_pair = (1 - β) · d_L^(-p) + β · d_C^(-p)
```

**Gradients:**
- Lifted portion: analytical, scaled by `(1 - β)`. Existing code unchanged.
- Clipped portion: finite differences through `clip ∘ T⁻¹`, scaled by `β`.
  - For each movable particle i, perturb each working-space coordinate by FD_EPS
  - Recompute `clip(T⁻¹(perturbed))`
  - Recalculate pairwise clipped energy against all other particles' clipped positions
  - Finite-difference the energy

Clipped positions computed once at the top (one `clip ∘ T⁻¹` per particle). FD adds 3 extra `clip ∘ T⁻¹` per movable particle — O(N) additional clip evaluations per iteration.

**Edge cases:**
- β = 0: clipped gradient skipped, lifted portion identical to V5.1
- β = 1: lifted gradient skipped, only FD gradient through clip
- In-gamut particles: clip = identity, clipped energy ≈ OKLab-distance energy

### 3. Optimization Loop

One line changes:

```ts
const beta = schedule.getMetricBlend(iteration);
const { forces: forceVecs, energy } = forces.computeForcesAndEnergy(particles, p, kappa, beta);
```

No other changes to the loop, projection, displacement, or convergence.

### 4. Wiring

`facette.ts` — no changes. `createForceComputer(lift, gamut)` already has both dependencies. `createAnnealingSchedule()` defaults include the β ramp.

No new files or abstractions.

### 5. Tests

**Existing tests:** All `computeForcesAndEnergy` calls gain `beta = 0` — signature alignment only, no logic changes.

**New tests in `energy.test.ts`:**
1. β=0 matches V5.1 (regression guard)
2. β=1 uses clipped distances — out-of-gamut particles produce different energy than β=0
3. β=1 in-gamut is near-identical to OKLab energy (clip = identity)
4. β=0.5 energy interpolates between β=0 and β=1
5. β=1 clipped gradient pushes out-of-gamut particles apart (correct sign)

**New tests in `optimization.test.ts`:**
6. Schedule β ramp: `getMetricBlend(0) === 0`, `getMetricBlend(rampEnd) === 1`, midpoint is 0.5

## Regression Analysis

| Seed config | Effect |
|-------------|--------|
| Muted (earth tones) | None. clip = identity, clipped energy = OKLab energy |
| Narrow hue, vivid | Minimal. Most particles in gamut. β blend has little effect. |
| Wide hue, vivid | Significant. Particles redistribute toward high-headroom positions. |

β=0 throughout reproduces V5.1 exactly. No regression for any configuration at default settings.
